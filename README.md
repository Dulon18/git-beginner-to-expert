## Overview

This course is a structured, 5-day deep dive into Git — the world's most widely used version control system. Whether you've never typed `git init` or you're ready to master rebasing and custom hooks, this course takes you from zero to confident Git user.

Each day builds on the last, with clear concepts, practical commands, guided exercises, and full solutions.

---

## Prerequisites

Before starting, make sure you have:

- ✅ A computer (Windows, macOS, or Linux)
- ✅ A terminal / command line
- ✅ [Git installed](https://git-scm.com/downloads) — verify with `git --version`

No prior Git knowledge required. Basic comfort with the terminal is helpful.

---

## Learning Outcomes

After completing this course, you will be able to:

- ✔️ Understand how Git works under the hood
- ✔️ Use branching and merging confidently
- ✔️ Collaborate with teams on GitHub
- ✔️ Resolve merge conflicts without fear
- ✔️ Recover lost work using reflog and reset
- ✔️ Use advanced workflows: rebase, cherry-pick, stash
- ✔️ Contribute to open source projects with pull requests
- ✔️ Write Git hooks and automate workflows

## Curriculum

### Day 1 — Git Basics
> *Start from zero, commit with confidence*

- What is version control and why it matters
- Installing and configuring Git
- Core concepts: repository, commit, staging area, `.git` directory
- File states: untracked, modified, staged, committed
- Essential commands: `init`, `add`, `commit`, `status`, `log`, `diff`
- Working with `.gitignore`

📁 [`Day-01-Git-Basics/`](./Day-01-Git-Basics/README.md)

---

### Day 2 — Branching & Merging
> *Work in parallel, merge with confidence*

- What branches are and why to use them
- Creating, switching, and deleting branches
- Merging strategies: fast-forward and three-way merge
- Resolving merge conflicts step by step
- Branch naming conventions and best practices
- Commands: `branch`, `checkout`, `switch`, `merge`

📁 [`Day-02-Branching-and-Merging/`](./Day-02-Branching-and-Merging/README.md)

---

### Day 3 — Remote Repositories & GitHub Collaboration
> *Go beyond your local machine*

- What remote repositories are
- Connecting to GitHub: `remote`, `push`, `pull`, `fetch`, `clone`
- Forking and pull requests
- Code review workflows
- Collaborating without stepping on each other's toes
- SSH keys vs HTTPS

📁 [`Day-03-Remote-Repositories/`](./Day-03-Remote-Repositories/README.md)

---

### Day 4 — Advanced Git
> *Power-user techniques for real teams*

- Interactive rebase: clean up commit history
- `git stash` — save work-in-progress instantly
- `cherry-pick` — apply specific commits across branches
- `git bisect` — find the commit that broke things
- `git blame` — trace every line of code
- Tags and releases: `git tag`

📁 [`Day-04-Advanced-Git/`](./Day-04-Advanced-Git/README.md)

---

### Day 5 — Expert Git
> *Automation, recovery, and open source mastery*

- `git reflog` — recover anything, even "deleted" commits
- Git hooks — automate checks with pre-commit, pre-push
- Submodules and monorepo strategies
- Advanced workflows: Git Flow, trunk-based development
- Contributing to open source: fork → branch → PR → merge
- Performance tips for large repositories

📁 [`Day-05-Git-Expert-Level/`](./Day-05-Git-Expert-Level/README.md)

## How to Use This Course

1. **Follow the days in order.** Each day builds on the previous one.
2. **Read the `README.md`** in each day's folder for concepts and explanations.
3. **Attempt the exercises** in `exercises.md` on your own first.
4. **Check `solutions.md`** if you get stuck — but try first!
5. **Use the Cheat Sheets** as quick references anytime.
6. **Complete the capstone project** at the end to solidify everything.

> 💡 **Tip:** Open your terminal alongside the course and run every command as you read. Hands-on practice is the fastest way to learn Git.
